# Enterprise Private Projects

本文件夹用于存放启蒙灯塔起源团队的企业私有项目，包括不开源的商业项目、内部工具、专有技术等。

## 文件夹说明

### 用途
- **商业项目**：不开源的商业应用和服务
- **内部工具**：团队内部使用的工具和脚本
- **专有技术**：保密的技术实现和算法
- **客户项目**：为特定客户开发的定制化项目

### 访问控制
- 仅团队核心成员可访问
- 不对外公开
- 遵守公司保密协议

### 文件组织

```
06_EnterprisePrivateProjects/
├── README.md                           # 本文件
├── [ProjectName1]/                      # 项目1文件夹
│   ├── README.md                       # 项目说明
│   ├── src/                            # 源代码
│   ├── docs/                           # 项目文档
│   └── ...
├── [ProjectName2]/                      # 项目2文件夹
│   └── ...
└── ...
```

### 命名规范

#### 项目文件夹命名
```
[ProjectName]/
```
- 使用有意义的英文名称
- 避免使用特殊字符
- 保持名称简洁明了

#### 文件命名
- 遵循项目内部的命名规范
- 保持一致性和可读性

### 注意事项

1. **不开源**：本文件夹中的所有项目均不开源
2. **保密**：遵守保密协议，不对外泄露项目信息
3. **权限**：只授予必要的团队成员访问权限
4. **备份**：定期备份重要项目文件
5. **清理**：及时清理不需要的文件和目录

## 项目管理

### 新项目创建流程
1. 在本文件夹下创建项目子文件夹
2. 添加项目 README.md 文件，说明项目背景和目标
3. 设置适当的访问权限
4. 在团队内部通知项目创建

### 项目归档流程
1. 项目完成后，评估是否需要归档
2. 归档项目时，添加 "_archived" 后缀
3. 更新项目状态和归档日期
4. 保留必要的文档和代码

## 联系方式

如有疑问或需要访问权限，请联系团队管理员。
