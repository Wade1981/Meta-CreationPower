## 技术要点

1. **端口管理**：
   - 根据API类型检查和分配端口，避免端口冲突
   - 当端口被占用时，自动查找可用端口

2. **进程管理**：
   - 为每个API服务启动一个独立的进程，避免资源竞争
   - 通过进程ID和端口号识别和停止对应的API服务

3. **状态管理**：
   - 保存API服务状态和进程ID到文件
   - 确保状态文件的更新与实际状态保持一致

4. **错误处理**：
   - 实现多种停止方式，提高服务停止的成功率
   - 验证服务是否真正停止，确保操作的可靠性

## 结论

所有API服务启动和关闭相关的优化已完成并验证通过。这些优化确保了：

1. **启动准确性**：启动单个API服务时不会错误地启动所有服务
2. **关闭准确性**：关闭单个API服务时不会错误地关闭所有服务
3. **后台运行**：API服务在后台运行，不阻塞终端
4. **状态显示**：启动后自动显示API服务的地址和端口
5. **可靠性**：多种停止方式确保服务能够被正确停止
