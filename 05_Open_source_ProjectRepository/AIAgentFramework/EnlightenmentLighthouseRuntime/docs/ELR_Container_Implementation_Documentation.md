# ELR 容器实现文档

## 1. 项目概述

ELR (Enlightenment Lighthouse Runtime) 容器是一个轻量级的容器运行时，基于 Go 语言开发，支持 Windows、Linux 和 macOS 平台。本实现文档详细描述了 ELR 容器的架构设计、核心功能和使用方法。

## 2. 架构设计

### 2.1 核心组件

- **Runtime**：容器运行时核心，负责容器的生命周期管理
- **Platform**：平台抽象层，处理不同操作系统的差异
- **Container**：容器实例，包含容器的配置和状态
- **NetworkManager**：网络管理器，处理容器的网络配置
- **TokenManager**：令牌管理器，处理身份验证和授权

### 2.2 目录结构

```
EnlightenmentLighthouseRuntime/
├── cli/             # 命令行接口
├── commands/        # PowerShell 命令模块
├── components/      # 组件
├── docs/            # 文档
├── elr/             # 核心运行时
├── examples/        # 示例
├── icons/           # 图标
├── micro_model/     # 微模型
├── models/          # 模型
├── output/          # 输出
├── platforms/       # 平台实现
├── python-portable/ # 便携式 Python
└── main.go          # 主入口
```

## 3. 核心功能

### 3.1 容器生命周期管理

- **创建容器**：创建新的容器实例，包括目录结构和配置
- **启动容器**：启动容器进程，设置资源限制
- **停止容器**：停止容器进程
- **删除容器**：删除容器实例和相关资源
- **列表容器**：列出所有容器实例
- **检查容器**：查看容器详细信息

### 3.2 文件系统隔离

- **根文件系统**：为每个容器创建独立的根文件系统
- **标准目录**：创建 bin、etc、home 等标准目录
- **网络配置**：为每个容器创建网络配置

### 3.3 进程管理

- **进程创建**：为容器创建独立的进程
- **进程隔离**：在 Windows 上使用 Job Objects 进行进程隔离
- **资源限制**：设置 CPU、内存等资源限制

### 3.4 网络隔离

- **网络配置**：为每个容器创建网络配置
- **IP 分配**：为容器分配唯一的 IP 地址
- **端口映射**：支持端口映射功能

### 3.5 命令行接口

- **create**：创建新容器
- **run**：创建并启动新容器
- **start-container**：启动容器
- **stop-container**：停止容器
- **list**：列出所有容器
- **delete**：删除容器
- **inspect**：检查容器详情

## 4. 平台实现

### 4.1 Windows 平台

- **进程隔离**：使用 Windows Job Objects 进行进程隔离
- **资源限制**：使用 Job Objects 设置 CPU、内存限制
- **文件系统**：创建独立的文件系统目录结构
- **网络配置**：为每个容器创建网络配置

### 4.2 Linux 平台

- **进程隔离**：使用命名空间进行进程隔离
- **资源限制**：使用 cgroups 进行资源限制
- **文件系统**：使用挂载命名空间或文件系统镜像
- **网络隔离**：使用网络命名空间

### 4.3 macOS 平台

- **进程隔离**：使用 sandbox 进行进程隔离
- **资源限制**：使用系统 API 设置资源限制
- **文件系统**：创建独立的文件系统目录结构
- **网络隔离**：使用网络配置

## 5. 使用方法

### 5.1 命令行使用

#### 创建容器
```bash
elr-container.exe create --name my-container --image ubuntu:latest
```

#### 运行容器
```bash
elr-container.exe run --name my-container --image ubuntu:latest
```

#### 启动容器
```bash
elr-container.exe start-container --id elr-123456789
```

#### 停止容器
```bash
elr-container.exe stop-container --id elr-123456789
```

#### 列出容器
```bash
elr-container.exe list
```

#### 删除容器
```bash
elr-container.exe delete --id elr-123456789
```

#### 检查容器
```bash
elr-container.exe inspect --id elr-123456789
```

### 5.2 配置文件

ELR 容器使用 YAML 配置文件，默认位于 `~/.elr/config.yaml`。配置文件示例：

```yaml
log_level: info
data_dir: ~/.elr/data
plugin_dir: ~/.elr/plugins
platform:
  windows:
    use_job_objects: true
    use_wsl: false
  linux:
    use_namespaces: true
    use_cgroups: true
  darwin:
    use_sandbox: true
    use_spctl: true
network:
  enable: true
  bridge: elr0
  subnet: 172.16.0.0/16
storage:
  enable: true
  driver: overlay
  base_dir: ~/.elr/storage
languages:
  cpp:
    enable: true
    runtime: /usr/bin/gcc
  python:
    enable: true
    runtime: /usr/bin/python3
  nodejs:
    enable: true
    runtime: /usr/bin/node
  java:
    enable: true
    runtime: /usr/bin/java
  go:
    enable: true
    runtime: /usr/bin/go
```
